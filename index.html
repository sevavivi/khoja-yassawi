Mausoleum of Khoja Ahmed Yasawi

Semantic Web, RDF and SPARQL Analysis of Islamic Cultural Heritage

Gulnara Kozmet & Sevara Abdulazizova

Explore Project View GitHub

Topic

This project focuses on the Mausoleum of Khoja Ahmed Yasawi, a UNESCO World Heritage Site located in Turkistan, Kazakhstan.

The monument is important not only as an example of Timurid architecture, but also as a religious, spiritual, and cultural site connected with Sufism and the Yasawiyya tradition.

Research Question

How can Wikidata for the Mausoleum of Khoja Ahmed Yasawi be enriched through RDF triples and knowledge graph modeling?

Methodology

1. Selecting the Topic
We selected the mausoleum because of its historical, spiritual, and architectural importance.
2. Exploring Wikidata
We examined the existing Wikidata entry and identified what information was already available.
3. Building SPARQL Queries
We created queries to retrieve structured data about the monument.
4. Identifying Gaps
We compared query results with historical and cultural knowledge.
5. Using LLMs
ChatGPT and Gemini helped generate and evaluate enrichment suggestions.
6. Creating RDF Triples
We proposed RDF-style triples to enrich the knowledge graph.

SPARQL Queries

SPARQL was used to investigate what information Wikidata already contains about the Mausoleum of Khoja Ahmed Yasawi.

Query 1 — Basic Information

SELECT ?property ?propertyLabel ?value ?valueLabel WHERE {
  wd:Q1133705 ?property ?value .
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en". }
}

Query 2 — Heritage Status

SELECT ?item ?itemLabel ?heritage ?heritageLabel WHERE {
  wd:Q1133705 wdt:P1435 ?heritage .
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en". }
}

Query 3 — Related Places

SELECT ?place ?placeLabel WHERE {
  ?place wdt:P131* wd:Q650439 .
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en". }
}

Identifying Knowledge Gaps

The current Wikidata entry contains basic historical and architectural information. However, several important aspects remain underrepresented.

Yasawiyya Sufi Order
The spiritual connection with the Yasawiyya tradition is not sufficiently modeled.
Pilgrimage Site
The mausoleum’s role as a pilgrimage destination is not fully represented.
Religious Significance
Its symbolic and spiritual meaning requires richer semantic description.
Related Heritage Sites
Connections with other religious heritage sites in Kazakhstan can be expanded.

Challenges

SPARQL Difficulty
Some queries returned no results because the expected relations were missing.
Ontology Mapping
Religious and cultural meanings are difficult to model with simple predicates.
LLM Verification
LLM outputs needed human checking to avoid hallucinated triples.
Knowledge Gaps
Central Asian Islamic heritage is still underrepresented in global knowledge graphs.

Conclusion

This project demonstrated how Semantic Web technologies and Large Language Models can support the enrichment of cultural heritage knowledge graphs.

By analyzing Wikidata through SPARQL, we identified missing information related to religious significance, pilgrimage practices, Sufi tradition, and connections with other heritage sites.

The project shows that LLMs can assist knowledge engineering, but their outputs must be critically verified before being used as structured data.
